# Interim Data Directory

This directory contains intermediate data files created during the data processing and feature engineering pipeline.

## Purpose

The `interim/` directory stores data that has been transformed from its raw state but is not yet in its final processed form. This allows for:
- Debugging data processing steps
- Caching intermediate results
- Supporting iterative development
- Enabling data pipeline checkpoints

## Current State

Currently empty (placeholder directory).

## Intended Use Cases

Typical interim data for the ecommerce recommender system:
- Partially cleaned event data
- User/item filtering results
- Intermediate feature matrices
- Sampled datasets for experimentation
- Aggregated statistics

## Data Flow

```
raw/ → interim/ → processed/
```

## Processing Pipeline

Interim data would be generated by:
- `src/recommender/data/dataset.py`: Data loading and initial cleaning
- `src/recommender/data/processors.py`: Data processing strategies
- `data_pipeline/pipeline.py`: Pipeline orchestration

## Models

Interim data supports model development by:
- Providing checkpoints for long-running processing
- Enabling quick iteration on feature engineering
- Supporting A/B testing of different processing strategies

## Metrics

Interim data may be used to compute:
- Data quality metrics
- Processing statistics
- Feature distributions
- Interaction sparsity measures
